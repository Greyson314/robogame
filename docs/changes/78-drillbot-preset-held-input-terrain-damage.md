# 78 — DrillBot preset, held-input drilling, terrain damage exemption

> Status: **shipped, machine gates green.** Three correlated changes
> from the second drill-playtest pass: a default chassis preset that
> actually ships with a drill, drilling tied to held left-click (not
> proximity auto-fire), and a chassis-on-terrain damage exemption so
> driving the drill into solid material doesn't shear the chassis off
> the drill cell.

## What changed

### 1) DrillBot preset

**[`GameplayScaffolder.cs`](../../Assets/_Project/Scripts/Tools/Editor/GameplayScaffolder.cs)**
gains `BuildDrillBotPlan` and `DefaultDrillBotPath`. Layout (forward
= +Z):

- 3 × 3 floor of cubes around a CPU at (0, 0, 0).
- One drill at (0, 0, 2), mounted on the +Z face of the front spine
  cube. So when the chassis drives forward, the drill is what hits
  terrain first.
- 4 wheels: two steerers at the front side-mounts (±2, 0, 1) and two
  drives at the back (±2, 0, -1). No weapon — left-click is the
  drill.

Wired into `BuildBootstrapPassA` as preset index 7 (after the
helicopter); preset array grew from 7 → 8 entries. Added to
`PresetBlueprintTests.PresetPaths` so the validator runs against it.

**Materialising the asset.** The `.asset` is generated by the
scaffolder; run `Robogame > Build Everything` (or whatever menu the
user has wired) to write the file. Until then,
`PresetBlueprintTests.Preset_PassesValidation` for the DrillBot
entry returns Inconclusive (same as any other freshly-added preset
before scaffolding).

### 2) Held-input drilling

**[`DrillBlock.cs`](../../Assets/_Project/Scripts/Voxel/DrillBlock.cs)**:

- New `IInputSource _input` field, resolved in `Awake` via
  `GetComponentInParent<IInputSource>()`. Mirrors the pattern
  `BombBayBlock` / `CannonBlock` / `ProjectileGun` use.
- The auto-poll `FixedUpdate` path AND the contact path
  (`HandleContact`) now both gate on `_input.FireHeld`. So the drill
  only carves while the player holds left-click.
- The public `Drill(zone)` method does NOT gate on input — it's the
  imperative-fire entry point for tests and scripted callers, and
  the held-input gating happens at the auto-trigger layer above it.
- Default drill radius bumped from 0.5 m → 0.8 m. A bit chunkier; a
  one-tick brush now carves roughly a 1.6 m wedge into terrain
  instead of a 1 m wedge. Tunable via the inspector.

**[`Robogame.Voxel.asmdef`](../../Assets/_Project/Scripts/Voxel/Robogame.Voxel.asmdef)**
adds a `Robogame.Input` reference so the asmdef sees `IInputSource`.

### 3) Terrain damage exemption

**[`MomentumImpactHandler.cs`](../../Assets/_Project/Scripts/Combat/MomentumImpactHandler.cs)**
adds an early-return on `OnCollisionEnter` whenever the other
collider's parent chain contains a `DigChunk`. Terrain has no HP,
and the drill block specifically needs to be in sustained contact
with the chunk surface to do its job — without this exemption, every
drill-vs-chunk contact rams the drill cell with reduced-mass kinetic
damage and chews through the chassis faster than the drill carves.

The exemption is broad ("any chassis block touching a chunk takes no
damage") rather than narrow ("only the drill cell is exempt") on
purpose, per the user's "at least for now" framing. Tightening to
drill-cell-only is a one-line edit when needed (compare
`contact.thisCollider`'s parent chain to find the touching
`BlockBehaviour`, suppress only if its `Definition.Id ==
BlockIds.Drill`).

## What's deferred

- **No PlayMode test for the terrain damage exemption.** Constructing
  a `Collision` object from C# isn't supported, so the test would
  need a full physics scene with synthesized contacts. The early-
  return is one branch; the visual playtest is the practical gate.
- **Tightening the exemption to drill-cell-only.** Acceptable for v1
  per "at least for now"; revisit if normal armor blocks scraping a
  dig zone in transit shouldn't be free.
- **Drill block visual.** The drill still uses the
  `CreatePrimitive(Cube)` fallback (no authored prefab); a real
  cone-shaped drill bit visual would help readability. Cosmetic;
  separate session.
